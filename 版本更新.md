# 版本更新

## 2026-06-05

本次更新把 RabiRoute 的配置结构从旧的单文件 / routes 口径整理成两个清晰的数据域，并同步了旧分支里仍然有价值的 WebGUI、Webhook、Codex 线程绑定和语音工作站能力。

### 配置结构

- 路由配置迁移到 `data/route/<配置名>/routeConfig.json`。
- 人格配置和消息模板规则迁移到 `data/roles/<角色名>/persona.md` 与 `data/roles/<角色名>/roleMessageConfig.json`。
- 删除公开示例中的旧 `examples/data/gateways.json` 和 `examples/data/roles/Rabi/routes.json`。
- 示例数据新增 `examples/data/route/main/routeConfig.json` 和 `examples/data/roles/Rabi/roleMessageConfig.json`。
- 配置检查脚本改为检查 `routeConfig.json` 和 `roleMessageConfig.json`。

### WebGUI

- 顶部操作栏改为固定悬浮，滚动页面时仍可刷新、启动 manager、新增路由配置、打开目录和保存配置。
- 路由配置和人格配置按 `data/route` / `data/roles` 两大数据域展示。
- 只有路由配置指向人格后，才显示人格配置和消息模板规则区域。
- 消息适配端设置改成集中式卡片和详情面板。
- 新增消息模板规则时，使用二级弹窗编辑；多条规则可通过标签切换。
- 所有配置区域补充“打开配置文件”入口。

### 路由与适配器

- Webhook 端口新增独立字段 `webhookPort`，避免和 NapCat WebSocket 的 `gatewayPort` 混用。
- Webhook adapter 使用 `WEBHOOK_PORT`，未配置时回退到 `GATEWAY_PORT`。
- NapCat adapter 增加活跃连接计数，避免多个 WebSocket 连接时误判断线。
- 语音转写默认模板更新为 FenneNote / Codex 本地语音输入场景，默认不自动转发到 QQ/NapCat。

### Codex 投递

- Codex App 和 Codex Desktop IPC 增强固定线程自动发现。
- 当旧状态绑定的线程名和当前配置不一致时，会清理旧绑定并记录错误信息，避免消息投递到错误线程。
- Codex Desktop IPC 保留批量投递和运行中 steer 逻辑。

### 文档与 Skill

- 文档统一改为 `data/route` / `data/roles` 新口径。
- 新增 `docs/voice-interaction-workstation.md`，说明 FenneNote、RabiRoute、角色对话和 OumuQ TTS 的公开安全接线方式。
- 新增 `skills/rabiroute-voice-workstation/SKILL.md`。
- 更新 `skills/create-rabiroute-persona`，避免后续继续生成旧 `routes.json`。
- 新增 NapCat 无值守与登录稳定性说明。

### 兼容性说明

- 本次不保留旧 `data/gateways.json` / `routes.json` 配置格式兼容。
- 旧分支已经归档为 `archive/*` tag；主分支按新格式继续演进。
- QClaw 分支差异过大，没有直接合并。后续如需要，应按当前主分支重新实现为可选 adapter 或插件。

### 验证

- `npm run build` 通过。
- `npm run check:config` 通过。
- 本地 manager 已用新构建重启，并确认 `8790`、`8789`、`8791` 监听正常。
- 发布前已做脱敏扫描，公开示例和文档使用占位值，没有提交运行期 `data/`、日志、token、cookie 或真实账号。
