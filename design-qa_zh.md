# 项目文档布局设计 QA

> 状态：历史 UI QA 记录。原参考图和实现截图没有提交到仓库，因此本文保留结论，不构成可复现的视觉证据包。

## 测试状态

- 路由：`/docs`。
- 页面：项目总览。
- 视口：1568 × 1100。
- 搜索：空。
- 视觉真源：用户提供但未提交的参考图。

## 布局结论

参考图使用顶部文档栏、左侧章节树、中间连续正文和右侧页内目录。实现保留 RabiRoute 应用外壳与浅色品牌主题，同时复现三栏阅读模型。

当时记录的桌面列宽为 `244px 764px 190px`。提高文档页网格选择器作用域后，右侧目录可以正常显示。

实现没有复制参考产品的深色主题、品牌、导航文案或正文内容。这些差异属于有意的产品边界，也不需要增加图片资产。

## 发现

- P3：RabiRoute 已有全局应用侧栏，文档可用宽度小于参考图，但三栏结构仍可使用。
- P3：功能详情继续用轻量事实卡展示真源、消费点、生效时机和副作用；其余章节采用连续正文。

最终记录中没有遗留可执行的 P0、P1 或 P2 问题。

## 保真检查

- 字体沿用现有中文字体栈，标题、导语、正文和页内导航层级清楚。
- 间距、分隔线、粘性导航和窄屏单栏降级符合预期阅读行为。
- 色彩保留 RabiRoute 浅色青绿色 token。
- 页面文案继续来自 RabiRoute 数据，没有替换成参考产品内容。

## 交互与运行检查

- 选择“边界规则”后只出现一个目标标题。
- 搜索 `Outbox` 后目录筛选到相关页面。
- 右侧锚点按当前页面动态生成。
- 桌面分组可以独立折叠子项。
- 选择“配置”分组会进入 Route 配置落地页。
- 移动端保留可访问的页面入口，只隐藏冗余箭头。
- 浏览器控制台没有应用错误。
- QA 当时 `npm run webgui:build` 通过。

## 历史与后续

首轮旧 `.docs-layout` 样式覆盖了三栏网格，导致右侧目录丢失。提高选择器作用域后恢复了预期列宽。

如果未来需要更接近参考图的阅读宽度，可以为 `/docs` 增加折叠全局应用侧栏的专注阅读模式。
