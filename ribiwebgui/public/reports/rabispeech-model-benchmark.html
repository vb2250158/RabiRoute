接口与工程结论

  • GET /v1/models 返回模型列表、安装状态、请求 schema、必填/可选参数与示例。
  • 普通 TTS/ASR 是直接 API，不接入 Agent；只有显式“提交 Route”才进入消息路由。
  • 角色声线由 data/roles/<RoleId>/voice/ 管理；播放跨会话、人格、模型使用全局 FIFO。
  • RTX 4080 SUPER 16 GiB 使用单 GPU worker 按需加载更稳妥。
  • 真实声卡流和服务重启恢复已通过；连续说话、噪声、回声、盲听和另一台电脑经 RabiLink 的公网往返延迟仍需用户验收。

Windows CUDA / DLL 记录

NVIDIA 驱动不等于 Python 推理所需的 cuBLAS/cuDNN DLL 已安装。RabiSpeech 使用 NVIDIA 官方 wheel 安装到插件私有目录，并只为服务进程追加 DLL 路径。FireRedASR2 使用 Windows 兼容的 Python 3.10、PyTorch 2.1.0+cu118、Transformers 4.51.3、NumPy 1.26.1、kaldi_native_fbank 1.22.3 与 setuptools<81。

完整方法、逐模型说明和限制见仓库 docs/rabispeech-performance-report.md。本报告只代表上方测试机与当次环境。
