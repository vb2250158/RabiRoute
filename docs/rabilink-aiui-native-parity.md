# AIUI 到手机 / 原生眼镜方案的功能等价清单

> 目标不是保留 AIUI 运行时，而是保留并改进用户可感知的全部能力。手机是完整客户端和可靠后端；眼镜是可开关的音频、HUD、相机与触摸板外设。

状态：`完成` 表示代码链已存在并有自动化或构建证据；`部分` 表示主链已实现但仍缺恢复/真机证据；`待补` 表示不能宣称等价。

| AIUI 能力 | 手机单独使用 | 连接眼镜 | 当前状态 | 等价实现 |
| --- | --- | --- | --- | --- |
| 配置完成后持续收音 | 手机前台麦克风服务 | 眼镜 App 自动采集 PCM | 部分 | Android 持续传有序 PCM，Rabi PC 统一 VAD/切句/ASR/声纹；需真机长时验收 |
| 单击触摸板立即提示 Agent 审阅 | 手机“提示 Agent” | 默认焦点单击“立即推送” | 完成 | 可靠 control 队列 -> `rabilink.review_request` -> 空闲启动/忙时 steer |
| 先存主机原始消息，再按 Route 策略投递 | 相同 | 相同 | 完成 | PC ASR 结果只写一次主机通用语音库；启用的 `rabilink` Route 再按 `hot/keyword` 进入人格，未订阅时只记录；普通 observation 仍可独立进入审阅器 |
| 连续反思和主动消息 | 相同 | 相同 | 完成 | PC `RabiActive`、reviewer 和 Outbox，不依赖客户端运行时 |
| ASR 失败恢复与静音切句 | PC RabiSpeech | PC RabiSpeech | 部分 | chunk 严格序号、Android 断流重建、PC 由 chunk 事件重置一次性 15 秒失活回收；公网抖动与真机恢复仍需验收 |
| 48 小时/2000 段离线上行 | 不适用于实时音频流 | 不适用于实时音频流 | 待补 | 文字/control/媒体仍有可靠队列；实时 PCM 断网期间不承诺缓存或补传 |
| 重复文本与 TTS 回声抑制 | 相同 | 相同 | 完成 | 2.5 秒重复抑制、12 秒回复回声抑制、播报时暂停/丢弃采集 |
| cursor 下行与离线 backlog | 手机服务 | 手机后端 | 部分 | cursor、TTS PCM 缓存、48 小时/2000 条已投递记忆；眼镜还需真实播放 ACK |
| TTS 失败让出队首并可重试 | 手机扬声器 | 眼镜扬声器 | 完成 | PCM/消息持久保存；连续三次失败让出队首，30 秒退避或用户显式重试 |
| 最新转写、回复和连接状态 | 手机持续会话卡 | 眼镜 HUD | 完成 | 同一后端事件同时更新手机运行态和眼镜文本协议 |
| 时间、电量、充电和版本 | 手机系统 UI | 眼镜 HUD | 部分 | 时钟/版本/真实 CXR 电量协议已接线，需真机状态刷新验收 |
| 暂停、继续、重试 | 手机开始/停止/重试 | 眼镜单行操作带 | 完成 | 持续聆听可暂停/继续，ASR/TTS 失败项有独立重试入口 |
| 自定义 ASR/TTS/语言/声线 | PC 配置 ASR；手机选择下行 TTS | 复用同一设置 | 完成 | ASR/VAD/语言归目标 PC RabiSpeech；手机只保留下行 TTS 模型与人格声线选择 |
| 复杂 PC 配置 | 手机远程 WebGUI | 不在眼镜堆叠后台表单 | 完成 | PC 仍是配置真源；手机打开 `/manage` 完成完整配置 |
| 自然语言配置助手与白名单动作 | 设置中的独立配置助手 | 可选眼镜语音入口 | 完成 | 配置不再共用聊天输入框；请求带标记进入所选 PC 人格，继续经过白名单、动作安全门和成功读回 |
| 首次绑定和多 PC 选择 | 手机应用 token + PC 选择 | 继承手机绑定 | 部分 | 不再使用眼镜 SN 领 token；需补扫码/短码和凭据轮换体验 |
| 安全云日志与离线补传 | 手机后端 | 眼镜错误经手机上报 | 完成 | 500 条/7 天私有队列；只上传粗粒度事件，不收正文、转写、token 或请求体 |
| 文本、图片、视频、音频和任意文件 | 完整双向聊天 | 眼镜照片与音频输入 | 完成 | 手机选择任意文件；PC 受 `allowedFileRoots` 约束下行；64 MiB/app 隔离附件和本地打开；纯附件下行不要求正文 |
| 两个常驻通知和普通消息通知 | 打开 App / 一键提示 | 共用手机通知 | 完成 | 前台服务通知 + 独立常驻提示；普通消息按会话聚合，通知直达详情，已读后清除 |
| Agent TTS 自动或手动播放 | 设置开关 | 共用设置 | 完成 | 关闭自动播放仍保存 WAV，点击聊天气泡手动播放 |
| 多 Rabi PC / 多路由人格 | 会话列表、独立详情和未读 | 继承当前人格 | 完成 | 只列出 RabiLink Route；逐会话草稿/已读，显式 `routeProfileId` 上下行，通知直达对应人格 |
| 开机后的消息恢复 | 后台恢复 cursor、队列与通知 | 打开 App 后恢复麦克风/眼镜 | 完成 | 开机使用 `dataSync` FGS，避开 Android 禁止从 boot 直接启动麦克风 FGS 的限制 |
| 手机无需眼镜独立工作 | 完整麦克风/扬声器/UI | 开关后增加眼镜 I/O | 完成 | 默认会话列表、聊天详情、前台服务、附件、通知和独立配置助手均不依赖眼镜 |

## 不允许的“迁移”解释

- “AIUI 暂停”不等于可以删除持续收音、触摸板审阅、离线队列、主动下行、状态 HUD 或配置助手。
- “手机可打开 WebGUI”只覆盖复杂可视配置，不自动覆盖自然语言配置助手。
- “音频已经发到手机”不等于已可靠投递；必须有稳定 ID、磁盘队列、cursor 和输出确认。
- “APK 可构建”不等于真机功能等价；完成状态还需要手机独立模式和眼镜模式分别验收。

## 完成门

代码层等价完成后仍不能用 APK 构建替代设备验收。表中标为“部分”的真实设备项必须在手机和眼镜上分别通过，才可宣称发布版“完整复刻 AIUI”。
