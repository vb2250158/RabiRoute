# 人格与消息规则

Route 决定消息怎么进入和投给哪个处理端；人格决定处理端以什么身份、背景和判断框架理解消息。两者分开保存，也可以独立复用。

## Route 和人格的边界

| 内容 | 归属 |
| --- | --- |
| 消息端、端口、处理端、工作目录、pipeline | Route |
| 指向哪个人格 | Route 的 `agentRoleId` |
| 人格正文、规则、计划、记忆和技能 | 人格目录 |
| 规则服务哪条 Route | 规则中的 `configName` |

一个人格可以服务多条 Route。修改人格正文或规则会影响所有绑定到它、且命中对应 `configName` 的 Route。

## 配置人格

打开“Rabi 人格”，在“指向人格”中选择已有角色。页面会显示 `persona.md` 预览、路由变量、默认规则和消息模板规则。

如果需要编辑完整正文，点击“打开人格配置”。运行语义文件不应机械翻译；改变语言、措辞和约束可能改变 Agent 行为。

## 一条规则包含什么

常用字段包括：

- **规则名称**：帮助使用者识别用途。
- **路由类型**：决定规则接收哪类事件。
- **消息匹配正则**：对群消息或文本做更窄匹配。
- **目标群号**：只限制指定群；留空表示不限群。
- **触发计划**：只对 heartbeat 规则产生定时事件。
- **Agent 消息包装模板**：给处理端补充当前场景的判断说明。

## 常用路由类型

| 类型 | 什么时候使用 |
| --- | --- |
| `private` | QQ 私聊 |
| `direct_at` | 群里直接 @ 当前账号 |
| `direct_reply` | 直接回复当前账号或角色参与的消息 |
| `indirect_reply` | 更宽的回复链观察，容易产生噪声 |
| `group_message` | 普通群消息，建议搭配窄正则 |
| `heartbeat` | 定时计划和手动验证 |
| `manual_trigger` | 只在用户或 API 明确触发时运行 |
| `role_panel_message` | 内置角色面板消息 |
| `voice_transcript` | FenneNote、小爱等语音转写 |
| `wecom_message` | 企业微信群消息 |
| `rabilink` | RabiLink 事件 |

界面会按当前 Route 已添加的消息端组织可选类型。没有选择任何类型可能匹配全部入口，通常不是安全默认值。

## 正则应该多窄

普通群消息不要使用空正则全量转发。先从能解释业务意图的关键词开始，例如：

```text
需求|报错|构建失败|提醒|请记录
```

正则只决定是否命中，不决定最终是否回复。人格模板仍应要求处理端先看上下文，区分新事实、任务、风险、普通确认和礼貌回应。

## 定时计划

`heartbeat` 规则支持：

- 每隔一段时间。
- 每天指定时间。
- 某一天指定时间。

包含 heartbeat 但没有计划时，规则仍可在日志诊断中手动触发。计划启用后会产生真实事件和投递记录。

## 模板写什么

模板适合补充“如何判断”，不需要重复拼完整消息。RabiRoute 已经注入事件、最近消息、人格路径、日志路径、知识索引和回复上下文。

工作型模板可以简短写成：

```text
先判断消息是信息、问题、任务、风险还是决策。
只有出现新增事实、阻塞或待办变化时才推进动作。
需要外部回复时，使用注入的 RabiRoute 回复接口。
```

不要只靠自然语言模板授予外部发送权限。实际回传仍受 pipeline 和消息端 policy 控制。

## 保存与验证

完成规则编辑后关闭对话框，再点击顶栏“保存配置”。下一条消息或下一次计划触发时生效。

当前 WebGUI 没有无副作用的 RouteDecision / AgentPacket 预览。验证规则请使用日志诊断的手动触发，并明确知道它会进入真实投递链。
