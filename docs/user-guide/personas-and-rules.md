# 人格与消息规则

Route 决定消息怎么进入和投给哪个处理端；人格决定处理端以什么身份、背景和判断框架理解消息。两者分开保存，也可以独立复用。

## Route 和人格的边界

| 内容 | 归属 |
| --- | --- |
| 消息端、端口、处理端、工作目录、pipeline | Route |
| 指向哪个人格 | Route 的 `agentRoleId` |
| 人格头像、正文、规则、计划、记忆和技能 | 人格目录 |
| 规则服务哪条 Route | 规则中的 `configName` |

一个人格可以服务多条 Route。修改人格正文或规则会影响所有绑定到它、且命中对应 `configName` 的 Route。

## 配置人格

打开“Rabi 人格”，在“指向人格”中选择已有角色。页面会显示 `persona.md` 预览、路由变量、默认规则和消息模板规则。

选择人格后，可以在同一张配置卡中设置或更换头像。支持 PNG、JPEG、WebP、GIF，单文件上限 5 MB。头像会跟随人格显示在选择器、Route 总览、语音人格选择和本地角色面板；没有头像时显示人格 ID 首字。头像属于人格目录，不需要为每条 Route 重复上传。

如果需要编辑完整正文，点击“打开人格配置”。运行语义文件不应机械翻译；改变语言、措辞和约束可能改变 Agent 行为。

## 在多台电脑间同步当前人格

选择人格后，“多电脑人格同步”会显示使用同一个 RabiLink 应用 token 的其它 PC。自动同步在后端运行，不要求一直打开页面：本机人格文件变化、其它 PC 上下线或 Relay 重连会触发一次 manifest 对账，优先走局域网，局域网不可达才经 Relay 受限中转。未完成的对账范围会落盘，断网和 Manager 重启不会把它忘掉；目标离线时等待连接事件，不按固定间隔查询业务数据。

页面可以查看自动对账状态，也可以点击“同步当前人格”立即执行。同步结果会区分拉取、推送、已一致、LAN/Relay 和冲突数量。普通文件双方都修改、删除与编辑并发时不会按最后写入者覆盖，而是进入“需要人工确认”：

- **保留本机**：保留当前文件，并尝试把这个决定发布回来源 PC。
- **采用对方版本 / 采用对方删除**：明确接受远端内容或删除意图。
- **手工合并**：让本机 Agent 使用 `use_merged` API 提交已经审阅的正文。

人格声纹关系的并发分支不在文件冲突窗口猜测谁是用户；回到下方“人格声纹归类”再次确认，新的关系事件会显式收敛分支。Relay 只负责发现和中转，不保存服务器端主人格；同步也不能替代独立备份或 Git/SVN。

## 归类当前人格的声纹

选择人格后，“人格声纹归类”会显示最近 24 小时的归类覆盖率、我的发言、其他人、未判断/冲突分段，以及当前人格已经保存的声纹关系。这里的“这是我”只代表当前人格对该处理主机声纹的明确解释；RabiSpeech 和 RabiRoute 主机不会替人格判断谁是谁，也不会默认把任何声纹设为用户。

对未解决声纹可以选择：

- **这是我**：把当前 `sourceHostId + voiceprintId` 关系记为当前人格认定的用户。
- **其他人**：明确记为非用户。
- **清除判断**：保留声纹关系事件，但取消 `isUser` 结论，重新回到未判断状态。

页面只请求统计、声纹缩写、时长、最后出现时间和关系，不请求或显示转写正文。新录音、本机关系修正和多电脑人格同步都通过事件触发一次刷新；事件流断线重连后会补查一次，不会固定间隔轮询覆盖率。关系最终写入人格自己的 `voice/voice-identities.jsonl`，并随人格文件夹按追加事件合并；多电脑冲突会明确显示，后续再次确认可收敛冲突分支。

第一次使用时，如果不透明声纹 ID 无法辨认，点击“标记下一段”，然后用准备归类的电脑、手机或眼镜，在尽量安静的环境里只让本人连续说一句。下一次录音事件完成后，本次新出现的未归类声纹会移到列表前面并标记“本次出现”。这个流程只是帮助缩小候选：它不会启动第二套录音、不会自动认人，也不会因为只出现一个候选就直接设为用户；如果现场同时有人说话，应只确认能够确定属于本人的声纹，或者重新捕获。

## 一条规则包含什么

常用字段包括：

- **规则名称**：帮助使用者识别用途。
- **路由类型**：决定规则接收哪类事件。
- **消息匹配正则**：对群消息或文本做更窄匹配。
- **目标群号**：只限制指定群；留空表示不限群。
- **触发计划**：只对 heartbeat 规则产生定时事件。
- **Agent 消息包装模板**：给处理端补充当前场景的判断说明。

## 常用路由类型

| 类型 | 什么时候使用 |
| --- | --- |
| `private` | QQ 私聊 |
| `direct_at` | 群里直接 @ 当前账号 |
| `direct_reply` | 直接回复当前账号或角色参与的消息 |
| `indirect_reply` | 更宽的回复链观察，容易产生噪声 |
| `group_message` | 普通群消息，建议搭配窄正则 |
| `heartbeat` | 定时计划和手动验证 |
| `manual_trigger` | 只在用户或 API 明确触发时运行 |
| `role_panel_message` | 内置角色面板消息 |
| `voice_transcript` | FenneNote、小爱等语音转写 |
| `wecom_message` | 企业微信群消息 |
| `rabilink` | RabiLink 事件 |

界面会按当前 Route 已添加的消息端组织可选类型。没有选择任何类型可能匹配全部入口，通常不是安全默认值。

## 正则应该多窄

普通群消息不要使用空正则全量转发。先从能解释业务意图的关键词开始，例如：

```text
需求|报错|构建失败|提醒|请记录
```

正则只决定是否命中，不决定最终是否回复。人格模板仍应要求处理端先看上下文，区分新事实、任务、风险、普通确认和礼貌回应。

## 定时计划

`heartbeat` 规则支持：

- 每隔一段时间。
- 每天指定时间。
- 某一天指定时间。

包含 heartbeat 但没有计划时，规则仍可在日志诊断中手动触发。计划启用后会产生真实事件和投递记录。

## 模板写什么

模板适合补充“如何判断”，不需要重复拼完整消息。RabiRoute 已经注入事件、最近消息、人格路径、日志路径、知识索引和回复上下文。

工作型模板可以简短写成：

```text
先判断消息是信息、问题、任务、风险还是决策。
只有出现新增事实、阻塞或待办变化时才推进动作。
需要外部回复时，使用注入的 RabiRoute 回复接口。
```

不要只靠自然语言模板授予外部发送权限。实际回传仍受 pipeline 和消息端 policy 控制。

## 保存与验证

完成规则编辑后关闭对话框，再点击顶栏“保存配置”。下一条消息或下一次计划触发时生效。

当前 WebGUI 没有无副作用的 RouteDecision / AgentPacket 预览。验证规则请使用日志诊断的手动触发，并明确知道它会进入真实投递链。
