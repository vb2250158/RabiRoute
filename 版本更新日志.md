# 版本更新

## 0.1.6 - 2026-07-08

### RibiWebGUI UI/UX

- 快速配置弹窗补齐保存中的 loading、保存失败提示和阻塞原因展示，避免保存失败时静默关闭或无反馈。
- 日志诊断页把手动触发区域中文化，并在触发成功或失败后直接显示结果，方便从同一页继续看最近日志。
- 人格消息模板规则页增加路由类型宽匹配、正则、heartbeat 计划和空模板说明的就地提示，帮助保存前发现高风险规则。

### 项目功能手册与 Agent 上下文

- 新增 `docs/project-function-map.md` 和 RibiWebGUI `/#/docs` 项目文档搜索页，按功能索引真源、消费点、生效时机、副作用、代码入口和设计边界，便于设计新页面、排障和 Agent 交接。
- 新增 `docs/persona-route-workbench-plan.md`，明确人格页后续应改造成“已绑定 route 的规则预览和诊断工作台”，而不是多人格智能命中或重复路由配置页。
- `personaConfig.json` 新增 `recentMessageLimit`，AgentPacket 会按 route/persona 配置附带最近消息摘要；默认 10 条，设为 0 可关闭，并补充配置读写和 packet 注入测试。
- RabiLink adapter 拆出为独立模块，Relay worker 与本地 replies 查询从通用 webhook adapter 中分离，保留 webhook/FenneNote/XiaoAi 的通用入口职责。

### Windows 桌面启动与完整包边界

- Windows 启动器、托盘 frozen 模式和打包脚本会检查后端 `dist/manager.js` 与 `ribiwebgui/dist` 是否缺失或过期，并按场景自动补构建。
- `docs/windows-launcher-and-packaging.md` 成为 Windows 桌面启动和完整打包的唯一真源，明确 `RabiRoute-Tray.exe` 只是完整桌面运行包里的托盘入口，不是单文件完整包。
- 打包脚本在无法覆盖正在运行的 `RabiRoute-Tray.exe` 时会输出 `RabiRoute-Tray.new.exe`，避免构建结果直接失败或误导用户。

### RabiLink 服务器控制台与远程 PC WebGUI

- Relay 管理入口统一为 `/manage`，登录后进入“RabiLink服务器控制台”；同一服务器支持多个账号，每个浏览器会话只保持一个当前登录账号，账号之间的应用、token、PC Rabi 连接记录和会话状态互相隔离。
- 服务器控制台展示当前账号下已连接的 PC Rabi，并提供“打开 PC WebGUI”入口：`/manage/<账号>/<RabiGUID>/#/routes` 会加载服务器托管的 RibiWebGUI 静态前端，配置读写请求通过 Relay worker 转发回对应 PC 的本机 Manager。
- Relay 新增 WebGUI 请求队列、worker 领取/回填接口、HTML/JS/CSS 路径前缀改写和响应体大小/等待时间配置；服务器不保存 PC 配置副本，PC Manager 仍是配置真源。
- 应用 token 继续在控制台卡片中默认显示预览，但登录后的“复制 token”按钮会随时复制完整 token，不再限制为创建或重新生成后的单次显示窗口。
- Relay 收紧应用 token 主从关系：旧公共 token 不再参与 RabiLink 任务鉴权，眼镜任务必须绑定到应用当前选择的 Rabi PC；没有选中 PC、绑定 PC 不存在或离线时直接返回错误，不再创建会被多台 PC 领取的无目标任务。
- 新增 `AGENT_TOKEN` OpenAPI 公开/模板导入版：插件级不写发布者 token，`submitRabiLinkTask` 用 body `token`、`getRabiLinkMessages` 用 query `token`，由智能体工具参数绑定到各自的 RabiLink 应用 token。
- RabiLinkMessage 插件描述补充 GitHub 使用说明链接，并在 OpenAPI `info.x-logo` 中指向 RabiRoute 图标，便于公开插件导入后补充介绍和图标。
- 服务器控制台新增按账号隔离的“最近日志”卡片：记录插件提交、下行轮询、PC Rabi 领取/回复、远程 WebGUI 请求等脱敏摘要，方便确认消息是否到达服务器且不暴露完整 token。
- 部署脚本会上传 `ribiwebgui/dist` 和必要静态资源，让服务器可以独立提供远程 PC WebGUI 前端；旧 `/manage/<账号>/<RabiGUID>/webgui/...` 路径保留为兼容入口。

### 手机 APK 与 RabiLink 绑定

- Android RabiLink SDK 增加服务器 mobile state、PC 列表、远程 Manager 配置、Route 列表和 Codex 绑定读写能力。
- 手机示例 APK 调整为“连接服务器 -> 选择 PC Rabi -> 选择 Route -> 选择 Codex 工作空间/会话 -> 保存绑定”的主流程，局域网扫描降级为备用诊断入口。
- 公开示例默认服务器地址使用 `https://rabi.example.com` 占位，避免把真实服务域名写入可提交源码。

### Rokid 眼镜测试 APK

- 眼镜端测试 APK 改名为 `Rabi Glass Test`，用纯黑背景实现 Rokid 光学透明，避免 Android translucent window 透出 launcher 或眼镜主菜单残留。
- 眼镜 HUD 收束为底部单行操作条：发送文本、Ping、状态、诊断、注册离线和清除离线。焦点由显式环形索引维护，选中项放大并居中，不再落到日志或标题上。
- 触摸板/方向键交互改为“上一个 / 下一个 / 确认”三动作模型：滑动只在松手时切换一次焦点，重复 key/touch 事件会去重，避免一次滑动跳过两个按钮。
- 启动眼镜 CustomApp 前会尝试关闭旧 CustomView，Activity 使用稳定启动模式并隐藏系统栏，减少重复启动和叠层残留。

### 本地验证

- `npm run build` 通过。
- `node --check scripts/rabilink-relay-server.mjs` 通过。
- `:app:assembleDebug` 通过，手机 APK 安装成功，眼镜端 APK 经重试安装成功并启动 CustomApp。
- `ar-glasses-gui-design` skill 校验通过。
- 线上 Relay 已部署并验证 `https://rabiroute.cottongame.com/health` 和 `/manage` 可访问，控制台页面已更新为“随时复制完整 token”逻辑。

## 0.1.5 - 2026-07-07

### NapCat WebUI 连接维护

- RibiWebGUI 的 QQ 实例卡片收束为“打开 WebUI”主动作：打开前会自动把发现到的 NapCat 实例加入配置、启用实例、补齐端口和登录信息，并在可修复时写入 OneBot HTTP/WS 配置。
- NapCat 健康检查补充 WebUI 可达但未读取到 QQ 登录态的诊断，提示用户先在 NapCat WebUI 完成扫码或前台登录；RabiRoute 只维护连接配置，不代替 NapCat/QQ 登录。
- WebGUI 构建目标调整为 `esnext`，保留当前前端语法输出，避免构建阶段错误降级现代语法。
- `.gitignore` 增加根目录 `.venv-tray/`，避免本机托盘打包虚拟环境进入提交候选。

### RabiLink Relay 直连

- RabiLink 主链路从“手机桥中转”调整为“电脑端 RabiLink worker 直连公网 Relay”：Rokid/灵珠插件提交任务到 Relay，Rabi 所在 PC 直接领取任务、投递给当前 RabiRoute/Codex 路由，并把增量回复写回 Relay。
- Relay server 新增服务器侧 `/admin` WebGUI：支持首次注册账号、账号密码登录、创建/启停/删除 RabiLink 应用、重新生成独立 `rbl_...` 应用 token；完整 token 只在创建或重新生成时显示一次。
- Relay 任务队列按应用 token 识别和隔离，Rokid/灵珠插件、电脑端 worker、task 消息查询和全局下行消息列表都只能访问对应应用的任务；旧 `RABILINK_RELAY_TOKEN` 当时仅作为兼容全局 token 保留，0.1.6 起已从 RabiLink 任务鉴权中废弃。
- WebGUI 的 `rabilink` 消息端新增 Relay 地址、token、设备名、领取等待和回复空闲超时配置，token 保存在本地私有 `data/route/<配置名>/adapterConfig.json`，不再写入公开示例。
- Relay worker 正式接口统一为 `/worker/tasks`、`/worker/tasks/<taskId>/messages`、`/worker/tasks/<taskId>/finish`，不再依赖 `/phone/tasks`。
- Rokid/灵珠侧继续使用全局下行 `/rokid/rabilink/messages` 拉取消息：不需要 taskId，拉到一句说一句，每条消息自带 taskId，直到 PC worker finish 后结束本轮。

## 0.1.4 - 2026-07-06

本次更新把 RabiLink 手机桥、Rokid/灵珠公网 relay、Android 探针工程和投递补偿能力合并到 RabiRoute 的公开工程结构中，并清理旧的 `android-band-probe` / `vela-band-probe` 示例命名。

### RabiLink 与 Rokid 桥接

- 新增 `rabilink` 消息端配置：支持 `/rabilink` 入站、独立 `rabilink` route kind、`rabilink-voice-transcripts.jsonl` 记录、RabiLink 专用 outbox 回复文件和 WebGUI 快速配置，不再把手机桥消息混作普通 `voice_transcript`。
- 新增 RabiLink Relay 服务脚本、部署脚本和 Rokid OpenAPI 示例，用于让 Rokid/灵珠云端插件经公网 relay 转交手机 RabiLink，再送入电脑 RabiRoute / Codex。
- Rokid/灵珠插件公开 OpenAPI 改为通过全局 `/rokid/rabilink/messages` 持续拉取下行消息，不再要求眼镜端按 `taskId` 拉取；每条下行消息自带 `taskId`，眼镜端应拉到一句说一句，直到 Rabi/Codex 侧完成并由手机桥 finish 后再结束。
- Relay server 默认长轮询等待时间统一调整为 60 秒，覆盖 Rokid 请求等待、task messages 拉取、全局下行消息拉取和手机桥领取任务，减少眼镜端频繁空轮询。
- 手机桥 finish 即使没有附带最终文本，也会唤醒 Rokid 全局下行长轮询，避免眼镜端在“已结束但无新句子”时继续显示思考中。
- 新增 Rizon 插件详情页“导入工具”专用 OpenAPI：`data/rabilink-relay/rokid-rabilink-tools-import.CURRENT.openapi.json`；它只导入 `submitRabiLinkTask` 和 `getRabiLinkMessages`，避免旧 taskId 调试工具影响正式智能体轮询。
- 新增 Rizon 插件详情页“导入工具”首选 Postman Collection：`data/rabilink-relay/rokid-rabilink-tools-import.CURRENT.postman.json`；用于绕过 Rizon OpenAPI 工具导入偶发的 URL 前缀转换误判。Postman 工具导入改为完整 HTTPS URL，不再使用 `{{base_url}}` 变量，并通过 `/rokid/rabilink/tools.postman.json` 提供 URL 导入入口。
- 新增 Android RabiLink SDK 与 `examples/android-rabi-link-probe/` 测试工程，统一承载小米健康、Rokid、RabiRoute manager 发现和 relay 桥接探针。
- RabiLink webhook 新增 `/rabilink/replies` 只读回包接口，Android 手机桥改从回调服务读取 Codex/RabiRoute 回复，避免局域网环境下依赖只绑定本机的 Manager 端口。
- Android RabiLink 探针增强常驻桥状态显示、桥接日志回显和扫描按钮状态；局域网扫描与公网 Relay 桥运行互相独立，扫描中会禁用重复点击但不阻塞正在运行的桥。
- 新增 `examples/rabi-link-vela-probe/`，替代旧 Vela 手环探针命名。

### 投递补偿与桌面可靠性

- Codex Desktop IPC 遇到 `no-client-found` 时增加唤醒、app-server fallback 和延迟重试记录，减少目标线程未加载时的消息丢失。

### 公开安全

- RabiLink 当前导入用 OpenAPI 改为本机 `data/rabilink-relay/` 运行文件并由 Git 忽略；仓库只提交 `examples/rabilink-relay/` 下的示例 JSON，避免公开提交真实域名、服务器地址和 token 相关运行配置。

## 2026-06-18

本次更新为远端 Agent 增加文件传输能力，让主控端可以随任务发送文件，远端也可以把产物文件带回本机。

### Remote Agent 文件传输

- `/api/remote-agent/tasks` 支持 `filePaths`、`files` 和 `attachments`；manager 会读取本机文件内容，随任务通过已认证的 WebSocket 控制通道发给远端 bridge。
- 远端 bridge 会把任务文件保存到远端运行期 inbox 目录，并在远端 Codex 任务提示中列出文件名、大小、sha256 和实际路径。
- 远端回调支持 `artifactPath`、`logPath` 和 `files[].path`；bridge 会读取远端本机文件内容回传给 manager。
- Manager 会把回传文件保存到 `data/remote-agent-files/<taskId>/`，并在任务事件 `savedFiles` 里记录本机保存路径、大小和 sha256。
- 文件传输默认不限制大小；如需给特定部署加保护，可通过 `REMOTE_AGENT_FILE_SINGLE_LIMIT_BYTES` 和 `REMOTE_AGENT_FILE_TOTAL_LIMIT_BYTES` 显式设置限制。
- Bridge 控制端口会验证 `127.0.0.1:<port>/health` 确实回到自己；如果本机 loopback 被旧服务占用，会自动顺延端口，避免远端 Codex 回调打到旧进程。
