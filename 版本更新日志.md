# 版本更新

## 0.1.15 - 2026-07-18

### Codex Desktop 稳定 ID 续投

- 修复第一条 Rabi 消息进入正确任务后，Desktop SQLite `title` 自动变成首条 prompt，导致第二条被误判为名称与 ID 不配对并创建同名任务的问题。
- 已保存绑定改为以“完整任务 ID + 规范化 workspace”为稳定身份；ID 存在、目录一致且未归档时直接续投，不比较可变标题。只有 ID 为空、非法或确实不存在时才按名称查找/创建。
- Rabi 设置中明确输入新名称仍会先清空旧 ID，再按名称查找或幂等创建；新增标题变化后的连续投递回归测试，并同步接入规范、验收合同、排障文档和 Agent 创建 Skill。

## 0.1.14 - 2026-07-18

### Codex Desktop 归档绑定保护

- 修复 Rabi 保存的 Codex Desktop 任务被归档后误判为“不存在”，继而自动创建同名替代任务的问题；按完整任务 ID 读取时现在会保留归档状态，并明确要求恢复原任务或重新选择。
- 已归档的固定绑定会返回冲突并停止投递，绝不进入同名查找或创建流程；普通下拉与名称查找仍只显示未归档任务，零匹配时才允许幂等创建。
- 新增 Manager API 与 Desktop 状态读取回归测试，并同步中英文接入规范、验收合同、排障文档和 Agent 创建 Skill；任务归档不再隐式改变 Rabi 的投递目标。

## 0.1.13 - 2026-07-18

### Codex Desktop 同名任务续投

- 修复保存 ID 失效、缺失或与名称不再配对时，同名同工作目录任务无法自动续投的问题：一个或多个精确候选会按 `updatedAt` 绑定唯一最新者，只有零匹配才允许创建新任务。
- 最新时间并列或全部时间不可解析时保持歧义并要求用户选择，不按数据库返回顺序猜测，也不会因此再创建同名任务。
- 新增 resolver 与 Manager API 回归测试，并同步中英文接入规范、验收合同、排障文档和 Agent 创建 Skill；Codex/ChatGPT Desktop 仍是唯一真实消息 owner，4510 生命周期与无备用 Runtime 边界不变。

## 0.1.12 - 2026-07-17

### Windows 安装包与 GitHub Release

- 新增 Windows x64 用户级安装器与便携 ZIP 构建链，内置固定版本 Node.js、Manager、RibiWebGUI、生产依赖和托盘入口；陌生电脑无需预装 Node.js 即可启动。
- Vue、Vuetify、Pinia、前端 Markdown 和图标字体归入开发依赖，发布 payload 不再重复携带已经编译进 WebGUI 的前端源码；生产依赖只保留 Manager 运行需要的包。
- 安装器默认放到 `%LOCALAPPDATA%\Programs\RabiRoute`，升级和卸载前都通过本机 Manager shutdown API 优雅停止运行态；payload 不包含 `data/`，覆盖安装和卸载都不会主动删除用户 route、人格与日志。
- `v*` tag 触发 GitHub Actions，在干净 Windows runner 上执行测试、配置检查、托盘构建、发布脱敏检查和打包 Manager 冒烟测试，再向 GitHub Release 上传 setup EXE、portable ZIP 与 `SHA256SUMS.txt`。
- 当前安装包尚未代码签名，README 与打包指南明确提示 SmartScreen 未知发布者警告和 SHA-256 校验；后续再根据真实发布频率评估代码签名、stable/nightly 通道和应用内更新。

### RabiPC 语音消息端与本地模型

- 新增独立、local-only 的 `rabi-speech` 插件和 RabiPC「语音服务」页面；顶部以 TTS / ASR 标签切换，导航位于「Rabi 人格」之后，投递 Route 只在用户显式启用时发生，普通 API 不进入 Agent。
- TTS 接入 ONNX-VITS、GPT-SoVITS、Qwen3-TTS 0.6B/1.7B、IndexTTS2 与 CosyVoice3；ASR 接入 faster-whisper tiny/small/large-v3-turbo、Qwen3-ASR 0.6B/1.7B、SenseVoiceSmall 与 FireRedASR2-AED。所有实现只允许已安装模型或本机回环 worker，云端语音 API 不在活动运行面。
- `GET /v1/models` 为全部 13 个模型返回安装/可用状态、请求 schema、必填/可选参数和示例；provider 接口仍可扩展，但远端请求不能下载模型、加载代码或修改 allowlist。
- TTS 支持直接按 Rabi 人格名调用、人格目录声线/缓存、多声线和跨 Route/会话/Agent 的主机级 FIFO；常驻 ASR 支持真实声卡、前置缓存、录音/转写双阈值、自适应底噪、静音切句、服务重启恢复和可选 Route 提交。

### RabiLink、公网边界与旧项目迁移

- RabiLink 作为 Manager 持有的系统级转接服务，使用通用应用 token 代理模型查询、TTS 与 ASR；本机服务保持回环监听，公网麦克风启停路径不在白名单内。
- OumuQ 与 FenneNote 进入停止维护归档，其 TTS 路由、人格声线、常驻转录和阈值能力由 RabiSpeech / RabiPC 语音消息端接替；活动 provider 不再依赖两个旧项目或付费云端 API。
- Rabi 语音相关 Skill 改用 RabiSpeech 与 `data/roles/<RoleId>/voice/`，Wiki 证据只提取声线、情绪、说话方式、台词示例与索引，并保持 `persona.md` 只读。

### 下载、性能报告与 Windows 运行时

- 新增逐模型下载/隔离运行时安装脚本与双语说明；CUDA/cuDNN DLL 安装在插件私有依赖目录，只修改服务进程搜索路径，不污染系统 PATH。
- 性能报告覆盖测试/建议硬件、模型体积、冷启动/预热、热请求、功能与限制，并提供 TTS 延迟、ASR 延迟、ASR 字符准确率三张柱状图和逐字对话样本；公网冷/热态与 Relay 重连结果分开记录。
- WebGUI 构建、配置检查、RabiRoute 203 项测试和 RabiSpeech 16 项测试通过；运行期 WAV、模型、日志、token、真实参考音、私有路径与本机配置仍保持忽略。

## 0.1.10 - 2026-07-16

### Codex 首次投递与人格初始化稳定性

- 新创建的 Codex Desktop 任务在首条真实消息前会有限等待 Desktop 只读索引出现同一个任务 ID；索引短暂延迟不再被误判为任务不存在，也不会因此重复创建同名任务。等待完成后，真实消息仍只通过 Desktop IPC 交给该任务 owner。
- 路由配置页新增“自动初始化会话”：先复用保存事务解析或幂等创建，并持久化“可见名称 + 完整任务 ID + workspace”，再通过角色面板和正式 AgentPacket 链把角色文件、记忆、计划及必读上下文投递到同一个 Desktop 任务。保存失败不投递，初始化首投失败只重试已经保存的任务 ID。
- 新增 Codex Desktop 专项验收合同，并把名称与 ID 重绑、按需扫描、唯一 owner、无 fallback 和 `127.0.0.1:4510` 独立生命周期写入 Agent 创建 Skill 与项目级 Agent 门禁。RabiRoute 不写 `CODEX_APP_SERVER_WS_URL`，不监听 4510，也不为了投递关闭或重启 Desktop。
- 回归覆盖新任务索引延迟、初始化保存/投递顺序和保存失败负例；全量 195 项测试、配置检查、Codex 契约检查、后端构建和 607 模块 WebGUI 生产构建通过。本机 Desktop 实机烟测中，已有会话投递与人格初始化均返回 `202`、消息在绑定任务可见，任务数量保持不变。

## 0.1.9 - 2026-07-16

### NapCat 群文件出站与受控构建产物交付

- NapCat Outbox 的本地群文件不再依赖普通消息 `file` 段，统一校验 `messageAdapterPolicies.napcat.allowedFileRoots` 后调用 OneBot `upload_group_file`，支持 APK 等大文件交付。
- 文件上传成功后可继续发送引用说明文本；若说明文本失败，结果仍保留“文件已上传”事实，避免自动重试时重复上传大包。返回结果新增 `sentFileName` 和可选 `sentFileId`，出站审计记录文件大小与真实路径。
- Agent 可以通过统一 Outbox 请求发送本地群文件；运行期路由按需开放 `file` 输出并配置构建产物白名单。只有实际 API 返回成功后才能宣称文件已发送，避免处理端因旧的文本权限假设直接拒绝交付。

### 示例 Rabi 全计划效率巡检

- 通用“一计划一会话任务持续追踪” Skill 增加强制效率不变量：计划未终态且没有有效阻塞时，绑定会话任务必须处于启动或运行状态；heartbeat 要枚举全部计划并把所有可推进但空闲、失败或 `systemError` 的任务立即续接或恢复。
- 阻塞不再允许只写“等待”。必须记录证据、能解除阻塞的负责人、待确认问题、联系结果、复核时间和解除后的动作；首次发现阻塞时主动找负责人确认，到期无结论时继续跟进或升级。
- 示例 Rabi 人格和 heartbeat 模板同步要求巡检结束时满足“可推进但空闲的计划数为 0”。运行期人格可以复制同一 Skill，并按自身已授权消息渠道执行负责人确认。

### RabiLink 眼镜云日志

- Relay 新增应用 token 鉴权的 `POST /api/rabilink/devices/logs` 批量入口，以及账号会话鉴权的 `GET /manage/api/device-logs` 查询入口。
- 管理控制台新增“眼镜云日志”，可按设备、来源、级别和关键词筛选，并显示应用版本、模式和会话。
- 日志按账号写入 `data/rabilink-relay/device-logs/<accountId>.jsonl`，支持稳定客户端 ID 幂等、保留上限和客户端/服务端双重脱敏。
- RabiLink AIUI `1.0.17` 新增非阻塞离线日志队列：最多保留 500 条、7 天，联网后每批 20 条补传；页面退出时先持久化，不等待网络。
- 云日志只包含运行状态、错误和安全 console 摘要；ASR 原文、配置需求、Agent 回复、token 和密码不上传。普通 AIUI 无权读取系统全局 logcat，系统级日志需未来的特权设备桥接入同一入口。
- 新增日志存储、鉴权、账号隔离、脱敏、去重和 AIUI 上传回归测试。

### Codex 自动会话解析与完整列表

### 运行时与状态可靠性

- 本机 Route 只扫描 Desktop 安装与 Desktop IPC 就绪状态，任务发现来自 Desktop 状态库，真实消息通过 Desktop owner start/steer。
- 项目根锁定的 `@openai/codex` 仅服务空任务元数据 bootstrap；独立 Remote Agent bridge 在远端设备上拥有并锁定自己的无人值守 stdio Runtime，两者都不得接管本机 Desktop。

- Agent 状态上报增加 gateway 进程 generation 与单调 sequence；Manager 只接受当前进程且顺序更新的状态，启动、停止和退出时清空旧状态，避免旧异步结果覆盖新状态。
- Codex 扫描提供 Desktop 任务候选；实际身份以 Desktop 状态中的完整 `codexThreadId` 和规范化工作目录共同确认并持久化。
