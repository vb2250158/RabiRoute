# 版本更新

## 0.1.3 - 2026-06-18

本次更新修正 NapCat WebUI 端口探测和健康检查回填行为，并把项目版本从 `0.1.2` 提升到 `0.1.3`。

### NapCat 健康检查

- NapCat 健康检查请求会携带 gateway id 和实例 id，让 manager 可以把检测结果准确回填到对应实例。
- Manager 会读取 NapCat `webui.json` 中的实际端口，在配置的 WebUI 地址不可达时尝试修正为本机实际端口。
- 健康扫描和单次健康检查会把修正后的 WebUI 地址写回 adapter 配置，并在响应 diagnostics 中说明自动修正结果。
- RabiWebGUI 会优先采用后端返回的 corrected WebUI URL，避免前端继续保存过期端口。

### Codex 线程发现

- Codex App 会遍历同名线程并选择第一个可读线程，减少最新同名线程不可读时误判为重复线程的情况。
- Desktop IPC fallback 调用恢复为普通 app-server 投递，由 Codex App 自己处理可读线程选择。

### 版本与验证

- 项目版本号同步更新为 `0.1.3`，覆盖 `package.json` 与 `package-lock.json`。
- `npm test` 通过。

## 2026-06-18

本次更新修复远端 Agent 在多网卡、VPN 或局域网地址公告不一致时偶发 `Remote Agent connection timed out` 的问题。

### Remote Agent 控制连接

- Rabi manager 扫描到远端 bridge 后，会优先使用 UDP discovery 响应的实际来源 IP 拼接 WebSocket 控制地址，而不是完全信任 bridge 自报的 host。
- 远端 bridge 控制端口仍来自 discovery 响应中的真实端口；端口自动顺延和 GUI 无需手动填端口的流程保持不变。
- 新增回归测试覆盖“bridge 自报地址和实际来源地址不同”时的控制 URL 生成，降低多网卡环境下 scan 成功但 connect timeout 的概率。

## 2026-06-10

本次更新继续把 RabiRoute 收束成“消息入口 + 角色面板 + 可回传 Agent”的调度层：Codex 适配器命名统一为 `codex`，角色面板成为内置本地消息端，定时触发支持规则级 schedules，NapCat 多实例状态和端口归属更可诊断，公开 Rabi 示例同步补齐计划/记忆知识包。

### 路由、Agent 与回传

- Agent 适配器类型从 `codexDesktop` / `codexApp` 统一迁移为 `codex`，旧配置在读取和保存时自动升级，Codex Desktop IPC 仍是默认已验证投递通道。
- 新增 `rolePanel` 消息端和 `role_panel_message` 路由类型，托盘角色面板可以把本地聊天消息投递给当前航线绑定的 Agent，并把 Agent 回复写回角色面板时间线。
- Agent 回复回传可以根据明确的 QQ 群/私聊目标或原始消息日志定位路由；NapCat 出站策略精简为 `outputEnabled` 和 `supportedOutputs`，旧的 `outputMode`、白名单和 pipeline 过滤字段不再作为新版发送判断。
- Manager 新增 Agent 运行态上报接口，Copilot CLI 和 Marvis 状态改为向 Manager 进程上报，减少读取陈旧 state 文件造成的误判。

### 定时触发、NapCat 与配置

- Heartbeat 定时触发支持每条通知规则配置 `schedules`，包括间隔触发、时间窗口内间隔触发、每日固定时间和一次性触发；触发记录会带上 route/rule/schedule 标识。
- Gateway 配置模型新增 NapCat 主实例解析、主实例字段同步和端口归属收集，端口自动分配会同时避开 manager、WS、HTTP、Webhook 等占用。
- NapCat health 检查增加 OneBot `get_status` 的 `online` / `good` 判断，并支持从 WebGUI 请求重启单个 NapCat 实例。
- 删除路由配置时只移除对应 `adapterConfig.json`，保留该 route 目录下的历史 JSONL 记录。

### WebGUI、托盘与示例

- RibiWebGUI 将角色面板显示为固定内置入口，补充路由删除、NapCat 重启、离线状态展示、dirty 配置保护和 heartbeat schedule 编辑/迁移入口。
- Windows 托盘窗口从只读计划/记忆看板升级为角色面板：左侧选择航线，主区域展示聊天、计划和记忆，支持发送文本和文件附件。
- 公开示例 Rabi 人格补齐计划索引、归档目录、多条沉淀记忆和近期记忆，用脱敏内容说明产品边界、配置源、消息链路、动作安全门、路由类型、WebGUI/托盘定位和开源发布卫生。
- README、配置文档、Agent 接口文档和 pipeline 文档同步改用 `codex`、新版消息端策略和角色面板/回传语义。

### 迁移说明

- 旧配置中的 `agentAdapters: ["codexDesktop"]` 或 `["codexApp"]` 会自动升级为 `["codex"]`。
- 旧 `messageAdapterPolicies` 中的 `outputMode`、`allowedGroups`、`allowedUsers`、`allowBroadcast`、`enabledPipelines`、`disabledPipelines` 字段会被忽略；需要控制发送时使用 `outputEnabled` 和 `supportedOutputs`。
- 旧 heartbeat 全局间隔仍会作为兼容 schedule 使用；新版建议在 heartbeat 通知规则上维护 `schedules`。
