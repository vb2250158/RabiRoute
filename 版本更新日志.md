# 版本更新

## 2026-06-07

本次更新把 RabiRoute 从单一 QQ/Codex 路由进一步扩展为“上下文路由 + 动作安全门”的多入口、多处理端框架，并补齐 Windows 桌面入口、小爱音箱桥接调研、Pipeline preset 和 RibiWebGUI 大幅重构。

### 路由与 Pipeline

- 新增 `pipelinePreset` / `pipeline` 配置，内置 `qq_chat`、`voice_chat`、`webhook_task` 三类输出意图。
- 模板变量新增输入端、输出端、TTS、回流控制和手动触发相关字段。
- 路由类型新增 `manual_trigger`，支持 manager API 主动触发指定规则。
- Webhook 语音事件新增设备 ID、设备名称、区域和 session 信息。
- 语音转写默认模板改为允许在授权充分时进入外发流程，避免把 voice transcript 一律视作只读输入。

### Agent 与输出适配

- Agent adapter 扩展到 `copilotCli`、`marvis` 和 `astrbot`。
- Copilot CLI 支持可配置二进制、工作目录、参数和超时。
- Marvis adapter 支持打开桌面端、复制提示词和通过 URL/桌面入口交接。
- AstrBot adapter 使用环境变量登录和缓存短期 token，不提交真实密码或 token。
- Manager 新增 Agent 扫描、Copilot 安装/登录/状态查询、FenneNote playback/reply 转发和 graceful shutdown API。

### RibiWebGUI

- WebGUI 页面改为控制台、消息适配器、Rabi 人格、日志诊断和路由目录导向。
- 路由配置页重构为更完整的消息入口、Agent 端、Pipeline、模板规则和手动触发配置界面。
- 总览页增强运行状态、连接诊断、快捷动作和路由选择。
- 人格模板页增强模板变量、规则编辑和人格配置展示。
- 运行日志页增强诊断信息和 manager 状态展示。
- 样式整体更新，改进密集配置界面的扫描性和移动端布局。

### Windows 桌面入口

- 新增并统一 Windows 启动入口到 `Start-RabiRoute-Tray.bat`。
- 新增 PySide6/Qt 托盘任务面板，只读展示当前任务、短期/长期计划、短期/长期记忆、任务列表和运行状态。
- 托盘面板通过 manager API 读取状态，可在 launcher 拥有 manager 生命周期时请求 graceful shutdown。
- 桌面层保持跨平台边界：Qt 面板、manager client、任务仓库、角色上下文仓库和生命周期控制保持可复用，Windows 只提供首个启动器。

### 小爱与外部设备

- 新增 `plugin-adapters/xiaoai-rabiroute`，提供 XiaoAI transcript bridge 和 speak placeholder。
- 新增 `examples/data/route/xiaoai/routeConfig.json` 和相关 `roleMessageConfig` 片段。
- 新增小爱拦截路由、LX06 刷机检查清单、红外遥控网关调研和 Home Assistant/BroadLink 选型说明。
- 本次不提交本地 `open-xiaoai` vendor 检出，只提交 RabiRoute 自己的桥接代码与文档。

### 文档、示例与发布卫生

- README 更新项目边界，明确 RabiRoute 保存可迁移上下文、路由决策、动作安全门和可观测记录。
- 新增 `docs/pipeline-presets.md`、Windows launcher/packaging 文档、WebGUI 构建 skill 和 RabiRoute 构建 skill。
- 示例 `.env.example` 增加 Copilot/Marvis 适配配置占位。
- `.gitignore` 增加 Python 虚拟环境、缓存、日志和本地 vendor 检出忽略规则。
- 发布前已清理本机绝对路径，公开内容只保留占位路径、localhost 示例和环境变量名。

### 验证

- `npm run build` 通过。
- `npm run check:config` 通过；本地运行期 `data/` 配置缺失项按脚本规则跳过，示例 route 和 Rabi role 配置校验通过。
- 脱敏扫描覆盖已修改与新增的待提交文件，未发现真实 token、cookie、密码或本机绝对路径。

### 追加更新

- Windows 入口合并到 `Start-RabiRoute-Tray.bat`，删除旧 `Start-RabiRoute-Windows.bat` 与 `scripts/start-rabiroute-windows.ps1` 分叉。
- 新增 `RabiRoute-Tray.spec` 与 `scripts/build-tray-exe.ps1`，支持 PyInstaller 本地打包根目录 `RabiRoute-Tray.exe`；exe 不提交进源码仓库，公开发布包暂不启用。
- 打包后的托盘 exe 在冻结模式下从自身目录定位项目根目录，并在 manager 未运行时自动启动 `node dist/manager.js`。
- 托盘退出时会优先调用 manager shutdown，并在拥有 manager 进程时直接终止本次托管进程，减少后台残留。
- Manager 配置拆分为 `data/route/<configName>/adapterConfig.json` 与 `data/roles/<RoleId>/personaConfig.json`，路由适配端和人格模板规则不再混在同一个文件里。
- WebGUI 路由配置页同步改用 `adapterConfig.json` 路径，并让 Copilot CLI 会话下拉只显示 Copilot session 名称。
- `.gitignore` 新增 `/build/`，避免提交 PyInstaller 临时目录和其中的本机路径分析文件。

## 2026-06-05

本次更新把 RabiRoute 的配置结构从旧的单文件 / routes 口径整理成两个清晰的数据域，并同步了旧分支里仍然有价值的 WebGUI、Webhook、Codex 线程绑定和语音工作站能力。

### 配置结构

- 路由配置迁移到 `data/route/<配置名>/routeConfig.json`。
- 人格配置和消息模板规则迁移到 `data/roles/<角色名>/persona.md` 与 `data/roles/<角色名>/roleMessageConfig.json`。
- 删除公开示例中的旧 `examples/data/gateways.json` 和 `examples/data/roles/Rabi/routes.json`。
- 示例数据新增 `examples/data/route/main/routeConfig.json` 和 `examples/data/roles/Rabi/roleMessageConfig.json`。
- 配置检查脚本改为检查 `routeConfig.json` 和 `roleMessageConfig.json`。

### WebGUI

- 顶部操作栏改为固定悬浮，滚动页面时仍可刷新、启动 manager、新增路由配置、打开目录和保存配置。
- 路由配置和人格配置按 `data/route` / `data/roles` 两大数据域展示。
- 只有路由配置指向人格后，才显示人格配置和消息模板规则区域。
- 消息适配端设置改成集中式卡片和详情面板。
- 新增消息模板规则时，使用二级弹窗编辑；多条规则可通过标签切换。
- 所有配置区域补充“打开配置文件”入口。

### 路由与适配器

- Webhook 端口新增独立字段 `webhookPort`，避免和 NapCat WebSocket 的 `gatewayPort` 混用。
- Webhook adapter 使用 `WEBHOOK_PORT`，未配置时回退到 `GATEWAY_PORT`。
- NapCat adapter 增加活跃连接计数，避免多个 WebSocket 连接时误判断线。
- 语音转写默认模板更新为 FenneNote / Codex 本地语音输入场景，默认不自动转发到 QQ/NapCat。

### Codex 投递

- Codex App 和 Codex Desktop IPC 增强固定线程自动发现。
- 当旧状态绑定的线程名和当前配置不一致时，会清理旧绑定并记录错误信息，避免消息投递到错误线程。
- Codex Desktop IPC 保留批量投递和运行中 steer 逻辑。

### 文档与 Skill

- 文档统一改为 `data/route` / `data/roles` 新口径。
- 新增 `docs/voice-interaction-workstation.md`，说明 FenneNote、RabiRoute、角色对话和 OumuQ TTS 的公开安全接线方式。
- 新增 `skills/rabiroute-voice-workstation/SKILL.md`。
- 更新 `skills/create-rabiroute-persona`，避免后续继续生成旧 `routes.json`。
- 新增 NapCat 无值守与登录稳定性说明。

### 兼容性说明

- 本次不保留旧 `data/gateways.json` / `routes.json` 配置格式兼容。
- 旧分支已经归档为 `archive/*` tag；主分支按新格式继续演进。
- QClaw 分支差异过大，没有直接合并。后续如需要，应按当前主分支重新实现为可选 adapter 或插件。

### 验证

- `npm run build` 通过。
- `npm run check:config` 通过。
- 本地 manager 已用新构建重启，并确认 `8790`、`8789`、`8791` 监听正常。
- 发布前已做脱敏扫描，公开示例和文档使用占位值，没有提交运行期 `data/`、日志、token、cookie 或真实账号。
