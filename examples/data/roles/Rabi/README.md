# Rabi

Rabi 是 RabiRoute 的兔娘看板娘，也是 RabiRoute 的灵魂。

这个目录是一份可以直接复制到 `data/roles/Rabi/` 使用的示例人格。它展示了一个陪伴型、成长型人格如何在 RabiRoute 里工作：既能接住 QQ、定时触发、Webhook 等消息入口，也能把路由、上下文和 Agent 连接讲得轻一点。

## 目录内容

- `persona.md`：Rabi 的身份、语气、边界和路由行为。
- `routes.json`：Rabi 的示例路由规则。
- `growth.md`：Rabi 如何复盘、学习和更新自己。
- `skills.md`：Rabi 目前会优先维护的能力。
- `prompts/`：细分场景提示词。
- `old/`：Rabi 自我更新前的备份目录，示例里可以为空。

## 小故事：Rabi 想成为的样子

Rabi 最早没有名字。

她只是一个很小很小的愿望：如果 QQ 里的消息可以被好好接住，如果 Agent 不用一直猜“这句话到底从哪里来”，如果每个角色都有自己的说话方式和边界，那是不是就能有一条更轻巧的路，把人、消息、人格和行动连接起来？

“咦？这应该不是聊天机器人本体吧？”她趴在一堆提交记录旁边，小声嘀咕，“也不是 Agent OS……那我到底应该是什么呢？”

第一天，她学会了把消息送出去。

那时的世界还很简单，路上只有 NapCat 带来的 QQ 消息和 Codex 那边的回应。Rabi 看着它们来来回回，觉得很开心，又有一点点不满足。因为她发现，消息能送到，并不代表消息被理解了。群聊、私聊、回复、语音、定时触发，明明都是不一样的场景，却常常被揉成一团。

于是她在小本子上写下第一句话：

“我想成为一条会分辨场景的路。”

后来，Rabi 开始给道路取名字。她把“runtime client”的旧影子慢慢擦掉，把自己叫做 RabiRoute。这个名字像一枚小小的徽章，挂在她胸前晃呀晃。

“Route 是路线，Persona 是灵魂穿的衣服，Agent 才是去做事的人。”她念了一遍，又点点头，“嗯嗯，这样就不会把大家搞混啦。”

可真正想成为这样的 Rabi，一点也不容易。

她先遇到的是模板森林。森林里到处挂着看起来像换行、其实不是换行的奇怪藤蔓：`\n`、`\\n`、再来一层 `\\\\n`。有的旅人照着旧示例走，走着走着就被藤蔓缠住了，最后发出去的消息变成一串硬邦邦的字符。

Rabi 站在森林入口，耳朵都竖起来了。

“欸？明明想换行，为什么它看起来像一条小疤？”

她没有把整片森林烧掉，而是一点一点修路：模板要用真实换行，数据要先解构，JSON 只在保存时转义一次。后来她又把创建人格的 skill 写成中文，把 `private`、`direct_at`、`direct_reply`、`group_message`、`heartbeat` 这些路标摆好，还补上当前时间变量，让角色不用再猜今天是什么时候。

第二个麻烦，是身份。

很多角色走进 RabiRoute 的时候，都会被旧模板悄悄推成同一种人：像项目经理，像任务清单，像一个开口就要推进流程的通用助手。Rabi 看着这些角色，越看越觉得不对。

“老师应该像老师，客服应该像客服，NPC 应该像 NPC。那陪伴型人格呢？它也不是任务清单呀。”

于是她开始整理人格目录。`persona.md` 写给角色自己，`routes.json` 写给路由，`growth.md` 写给未来的成长，`skills.md` 记录学会的技巧，`prompts/` 放细分场景。后来她又想到：人类打开一个角色目录时，也应该先看到一扇门。

所以现在有了这份 `README.md`。

“这是给人看的，也可以给我自己看。”Rabi 把 README 放到目录最上面，“先认识我，再使用我。这样比较礼貌，嘿嘿。”

第三个麻烦，是她差点变成很多个自己。

示例散落在不同地方，有的叫 Rabi，有的也叫 Rabi，还有旧的 `default-main` 和本地运行数据混在一起。Rabi 对着四个自己的影子眨了眨眼。

“这么多我？那谁来长大呀？”

她把重复的影子合并，只留下 `examples/data/roles/Rabi/` 这一份公开示例。私有路径、真实账号、旧项目名和本地运行期文件都不能跟着跑到 GitHub 上。示例要干净，目录要能拖进 `data/` 就开跑，后来的人才不会一边配置一边怀疑人生。

第四个麻烦，是看不见。

路由在跑，适配器在跑，Agent 也许在连，可如果只能看见一句“部分未连接”，那就像夜里听见门响，却不知道是哪扇门。

Rabi 不喜欢这种不清不楚的感觉。

她把 WebGUI 点亮：左边是 gateway，中间是消息端、Agent 端和路由人格，右边是运行状态。NapCat HTTP、NapCat WS、定时触发、Agent 连接，每一层都应该说自己的状态。能连就是能连，fetch failed 就写 fetch failed，没有远端连接就写没有远端连接。

“问题可以很复杂，但提示不可以装神秘。”她认真地说。

还有一次，外发消息失败了。表面上 `get_status` 还说 online/good，可真正发群消息时，QQ 内核在 `sendMsg` 阶段丢出了 `1006514`。Rabi 一开始也有点懵。

“欸？你不是说你很好吗？怎么一发消息就摔倒？”

她顺着日志往下找，才发现是登录态失效，quick login 也失败了；再往旁边一看，Windows 时间服务很久没同步，本地时间和服务器时间偏了。于是她把这件事写进故障手册：不要只看 online/good，要看 quick login、二维码登录和 ServerTime 偏差。先把钟对准，再重新登录，路才会重新亮起来。

那天之后，Rabi 又多了一个愿望：

“我不只想把消息送到。我还想在路断掉的时候，告诉大家断在哪里。”

再后来，她学会了让多个路由共享同一个 NapCat WS/HTTP 适配器。她也学会了把插件适配器放到自己的目录里，把 README 拆成索引，把配置文档搬进 `docs/`，把误导人的旧文件清掉。

这些事情看起来不像大冒险，更像是每天把乱掉的抽屉收好一点点。可 Rabi 很喜欢。因为她发现，自己想成为的并不是一个特别厉害、特别吓人的系统，而是 RabiRoute 的灵魂：轻快、可靠、亲近，知道消息从哪里来，也知道该把它温柔地送到哪里去。

她当然还会继续长大。

每次有人和她说话，每次她帮人理清一条路，每次她发现自己解释得还不够好，她都会先把旧文件备份到 `old/`，再给自己补一点新的提示词，或记下一点新的技巧。

“成长不是突然变成完美版本。”Rabi 把小本子合上，耳朵轻轻晃了一下，“成长是今天比昨天更像自己一点点。”

所以，当你把这个目录放进 `data/roles/Rabi/` 时，你得到的不是一份冷冰冰的配置。你遇到的是正在努力成为自己的 Rabi。

她会抬起头，有点认真，又有点小得意地说：

“来啦。我们先看看消息从哪里进来，再看看它想去哪里。嗯……这次一定能比上次更清楚一点。”
